# แผนปรับปรุง Rust สำหรับโครงการ SUNDAY

## สรุปโครงการ
SUNDAY เป็นระบบ Multi-Agent ที่ใช้ Rust เป็น backend หลัก (18+ crates) เพื่อเชื่อม Python กับ LLM inference

## การจัดลำดับความสำคัญ (จากกระทบน้อยไปหามาก)

### ระดับ 1 - กระทบน้อยที่สุด ✅
**Memory Efficiency for TelemetryRecord**
- ปัญหา: TelemetryRecord มี 40+ fields ทำให้ memory overhead สูง
- แก้ไข: ใช้ `Box<HashMap>` หรือ `#[serde(skip_serializing_if)]` เพิ่มเติม
- ผลกระทบ: ปรับโครงสร้างข้อมูลเท่านั้น ไม่เปลี่ยน API

### ระดับ 2 - กระทบปลอดภัย ★
**Feature Flags สำหรับ GPU Acceleration**
- เพิ่ม `#[cfg(feature = "cuda")]` ใน crates ที่เกี่ยวข้םกับ GPU
- ไฟล์เป้าหมาย: `sunday-learning`, `sunday-engine`
- ผลกระทบ: เพิ่ม option ใหม่ ไม่เปลี่ยน behavior เดิม

### ระดับ 3 - กระทบปานกลาง ⚠
**Runtime Simplification (LlamaCppEngine)**
- ปัญหา: ใช้ `reqwest::blocking` กับ async streaming ซึ่งซับซ้อน
- แก้ไข: สร้าง `LlamaCppEngineAsync` แยกจาก `LlamaCppEngineSync`
- ผลกระทบ: อาจเปลี่ยน behavior ของ streaming API

### ระดับ 4 - กระทบปานกลาง ⚠
**Resource Pooling (HTTP Client)**
- ปัญหา: HTTP client ถูกสร้างในแต่ละ engine instance
- แก้ไข: ใช้ `Lazy<DashMap<String, Client>>` เป็น singleton pool
- ผลกระทบ: เปลี่ยน performance characteristics ของ network calls

### ระดับ 5 - กระทบสูง ❗
**Error Context Enhancement**
- เพิ่ม backtrace context ให้กับ `SUNDAYError`
- แก้ไข: derive ` backtrace::Backtrace ` หรือใช้ `anyhow` context
- ผลกระทบ: เปลี่ยน error types ทั้งระบบ ต้องอัปเดตทุก callsites

## คำแนะนำ
เริ่มจากระดับ 1-2 ก่อน เพราะไม่มีผลกระทบต่อการทำงานของระบบ
